### 예상 원인

`type:"window"`는 내부적으로 `window.open`을 사용하므로, 브라우저 환경 설정에 영향을 받습니다. **특정 사용자 PC에서만** 발생하는 경우 아래 항목을 순서대로 확인해 주시기 바랍니다.

#### 1. Edge IE모드 사이트 목록(SiteList) 설정 누락

SP2를 사용 중이시라면 Edge IE모드로 운영할 가능성이 높습니다. `type:"window"`로 열리는 팝업 URL이 Enterprise Site List에 등록되지 않은 경우, 팝업이 Edge 모드로 전환되면서 경로를 찾지 못하는 오류가 발생할 수 있습니다.

**확인 방법:**
- Edge 주소창에 `edge://compat/enterprise` 입력하여 Site List 확인
- 팝업 URL(전체 경로)이 목록에 포함되어 있는지 확인
- 포함되어 있지 않다면 IT 관리자에게 해당 URL 추가 요청

#### 2. 보안 영역(Zone) 설정 차이

IE모드는 Internet Explorer의 보안 영역 설정을 따릅니다. 특정 사용자 PC의 보안 영역 설정이 다를 수 있습니다.

**확인 방법:**
- 제어판 > 인터넷 옵션 > 보안 탭
- 해당 사이트가 "신뢰할 수 있는 사이트" 또는 "로컬 인트라넷"에 등록되어 있는지 확인
- 정상 동작하는 사용자 PC의 설정과 비교

#### 3. 팝업 차단 설정

`type:"window"`는 내부적으로 `window.open`을 사용하므로 팝업 차단에 의해 에러가 발생할 수 있습니다.

**확인 방법:**
- Edge 설정 > 쿠키 및 사이트 권한 > 팝업 및 리디렉션에서 해당 사이트 허용
- IE 인터넷 옵션 > 개인 정보 > 팝업 차단 설정도 함께 확인

#### 4. 상대 경로 vs 절대 경로 문제

openPopup의 url 파라미터가 상대 경로인 경우, IE모드에서 base URL 해석이 달라 경로를 찾지 못할 수 있습니다.

```javascript
// 상대 경로 (문제 가능성 있음)
$w.openPopup("/ui/popup/sample.xml", { type: "window" ... });

// 절대 경로로 변경 테스트
$w.openPopup(location.protocol + "//" + location.host + "/ui/popup/sample.xml", { type: "window" ... });
```

#### 5. URL 인코딩 문제

IE모드에서는 URL에 한글 또는 특수문자가 포함된 경우 인코딩 처리가 Edge 표준모드와 다르게 동작할 수 있습니다.

### 추가 확인 — 간헐적 발생(됐다가 안됐다 하는 경우)

특정 사용자가 됐다가 안됐다 하는 경우, 고정 설정 문제보다 아래 원인이 유력합니다.

#### 6. 팝업 차단 타이밍 이슈 (가장 유력)

`type:"window"`는 내부적으로 `window.open`을 사용하는데, 브라우저는 **사용자 클릭 이벤트 직후**에만 팝업을 허용합니다. 클릭 → 비동기 처리(Ajax 콜백, setTimeout 등) → openPopup 순서로 호출하면, 브라우저가 때에 따라 차단할 수 있습니다.

```javascript
// 문제 가능성 — Ajax 콜백 안에서 openPopup
scwin.btn_onclick = function() {
    $.ajax({
        success: function(data) {
            $w.openPopup(url, { type: "window" }); // 비동기 콜백 → 팝업 차단 가능
        }
    });
};

// 해결 — 클릭 직후 즉시 호출
scwin.btn_onclick = function() {
    $w.openPopup(url, { type: "window" }); // 클릭 이벤트 내 동기 호출
};
```

#### 7. Edge IE모드 세션/프로세스 전환 타이밍

Edge IE모드에서 window.open 호출 시 IE 프로세스가 별도로 생성되는데, PC 부하 상태나 메모리 상황에 따라 프로세스 전환이 늦어지면서 경로를 못 찾는 경우가 있습니다.

#### 8. 브라우저 캐시/세션 만료

세션이 만료된 상태에서 팝업을 열면 로그인 페이지로 리다이렉트되면서 경로 오류처럼 보일 수 있습니다.

보다 정확한 원인 파악을 위해 아래 정보를 추가로 확인 부탁드립니다.

2. **에러 메시지 전문** (스크린샷 또는 텍스트)
3. **오류 발생 시점의 패턴** (장시간 사용 후 발생하는지, 특정 시간대에 발생하는지)
4. **openPopup 호출 코드** (url, 옵션 파라미터 전체)
5. **사용 중인 웹스퀘어 엔진 버전** (미리보기에서 Ctrl+마우스 우클릭 > 디버그 메뉴 > Version 정보)
