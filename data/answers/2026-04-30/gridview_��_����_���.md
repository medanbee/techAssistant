GridView 좌측 헤더 구현 방법과 관련하여 유사 사례 기반으로 안내드립니다.

### 원인 분석

GridView 컴포넌트는 리스트 형태의 데이터를 빠르게 표현하는 데 최적화된 컴포넌트로, 구조적으로 좌측에 고정된 헤더 영역을 만드는 기능을 공식적으로 지원하지 않습니다.

### 해결 방법

원하시는 UI를 구현하기 위한 두 가지 방법을 안내해 드립니다.

1.  **`tableLayout` 컴포넌트 사용 (권장)**
    상단 헤더와 좌측 헤더를 동시에 가지는 복잡한 표 구조는 `GridView`보다는 `tableLayout` 컴포넌트를 사용하여 구현하는 것이 더 적합합니다. `tableLayout`은 HTML의 `<table>`과 유사하여 자유로운 레이아웃 구성이 가능합니다.

2.  **`GridView` 기능을 이용한 유사 구현**
    `GridView`를 반드시 사용해야 하는 경우, 아래와 같은 기능들을 응용하여 좌측 헤더와 유사한 효과를 낼 수 있습니다. 참고 사례에서는 아래와 같은 방법을 제시하고 있습니다.

    *   **`mergeCells` API 활용**: 스크립트를 사용하여 동적으로 특정 범위의 셀들을 병합하여 좌측 헤더처럼 보이게 할 수 있습니다.
        ```javascript
        // 예시: 0번 행부터 3번 행까지, 첫 번째(0번 인덱스) 컬럼을 병합합니다.
        gridView1.mergeCells(0, 0, 3, 0);
        ```
        ※ 위 코드는 유사 사례 기반 참고용입니다. 실제 동작 확인 후 적용해주세요.

    *   **Body 구조 직접 설계**: GridView의 Body 영역에 `w2:row`를 직접 사용하여 시각적으로 좌측 헤더처럼 보이도록 레이아웃을 구성하는 방법을 고려할 수 있습니다.

### 추가 확인 사항

*   안내된 `mergeCells` API 등을 이용한 방법은 GridView의 공식적인 좌측 헤더 기능이 아닙니다.
*   GridView는 스크롤 시 동적으로 행(Row)을 생성하는 방식이므로, 이러한 방법들이 스크롤, 정렬, 필터링 등 다른 기능과 함께 사용할 경우 예상치 못한 동작을 유발할 수 있습니다. 적용 전 충분한 테스트를 통해 동작 안정성을 확인하시기 바랍니다.
