GridView 파일 다운로드 및 업로드 시 'TypeError: (reading 'substring')' 오류 발생과 관련하여 안내드립니다.

`Uncaught TypeError: Cannot read properties of undefined (reading 'substring')` 오류는 일반적으로 스크립트가 특정 문자열(String) 타입의 데이터를 예상했으나, `undefined` 값이 전달되어 `substring` 함수를 실행하지 못할 때 발생합니다.

파일 업로드 시 "파일이 암호화되어 있습니다." 메시지가 함께 나타나는 점으로 미루어, 일반적인 GridView 기능 오류보다는 **파일 암복호화 관련 로직** 또는 **서버와의 통신 과정**에서 문제가 발생했을 가능성이 있습니다.

### 원인 분석

| # | 원인 | 설명 |
|---|---|---|
| 1 | 서버 응답 데이터 형식 오류 | 파일 업로드/다운로드 요청 시 서버로부터 예상치 못한 형식의 응답(에러 메시지, HTML 페이지 등)을 받아, 클라이언트 스크립트가 이를 처리하는 과정에서 `undefined` 참조 |
| 2 | 파일 암복호화 모듈 충돌 | 클라이언트/서버 측 보안 솔루션(DRM 등)이 동작하면서 스크립트가 정상 파일 정보를 받지 못해 `substring` 호출 실패 |
| 3 | 엔진 빌드 이슈 | 빌드 5.0_5.5219A (2024-07-24)는 비교적 옛 빌드. 관련 핫픽스가 누락되었을 수 있음 |

### 해결 방법

현 시점 정보만으로 명확한 해결 방법을 단정하기 어렵습니다. 아래 '추가 확인 사항'으로 문제 범위 좁혀나가는 것을 권장드립니다.

### 추가 확인 사항

#### 1. 브라우저 개발자 도구 확인
- 오류 발생 시점에 F12 열어 **콘솔(Console)** 탭의 전체 오류 메시지 + 스택 트레이스 확인
  - 오류 발생한 스크립트 파일명과 라인 번호 → 원인 추적
- **네트워크(Network)** 탭에서 파일 업로드/다운로드 관련 요청 확인
  - HTTP 상태 코드 및 응답 본문(Response) 정상 여부

#### 2. 파일 암복호화/보안 솔루션 확인
- 현재 시스템에 별도 보안 솔루션(DRM, 파일 암복호화 등)이 적용되어 있는지
- 적용되어 있다면, 잠시 비활성화 후 테스트 시 동일 문제 발생하는지

#### 3. 관련 소스 코드 공유
- GridView의 엑셀 다운로드 및 파일 업로드를 실행하는 소스 코드 일부 공유 가능 여부
  - 예: `gridView.advancedExcelDownload()`, `upload.upload()` 등 호출부

#### 4. 엔진 버전 업그레이드 검토
- 현재 빌드(2024-07-24) 이후 관련 패치가 적용되었을 가능성
- 가능하다면 SP5 최신 빌드로 업그레이드 후 재테스트 권장
