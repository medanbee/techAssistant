안녕하세요.
인스웨이브 기술지원팀 이단비 프로입니다.

엑셀 업로드 시 작은따옴표(') 이스케이프 변환 현상과 관련하여 확인 후 답변드립니다.

## 원인 분석

엑셀 업로드 시 WebSquare 내부에서 작은따옴표(')를 문자열 충돌 방지를 위해 자동으로 이스케이프 처리(`\'`)하는 동작입니다. 이는 오류가 아닌 **WebSquare 내부 설계 동작**입니다.

- 엔진 기본 설정으로 `dataRefine` 속성이 **true**로 되어 있어, gridView 엑셀 업로드 시 작은따옴표(')를 자동 치환합니다.
- 이 설정은 과거 고객사 요청으로 2019년에 추가된 속성으로, 작은따옴표가 포함된 데이터가 정상 출력되지 않는 현상을 방지하기 위한 것입니다.
- 버전에 따라 `'` → `` ` ``(백틱) 또는 `'` → `\'`(역슬래시) 형태로 치환될 수 있습니다.

## 해결 방법

### 방법 1: dataRefine=false 설정 (전역 적용)

`websquare.xml`에 아래 설정을 추가한 후 WAS 재기동합니다.

```xml
<websquare>
  <excel>
    <upload>
      <dataRefine>false</dataRefine>
    </upload>
  </excel>
</websquare>
```

- **장점**: 설정 한 줄로 간단히 적용
- **주의**: 전체 데이터 정제 처리가 비활성화되므로, 다른 특수문자 치환에도 영향이 있을 수 있음

### 방법 2: dataConvertor 구현 (선택적 치환)

`ICellDataProvider` 인터페이스를 구현하여 특정 문자만 선택적으로 치환할 수 있습니다.

**1) websquare.xml 설정**

```xml
<websquare>
  <excel>
    <upload>
      <dataConvertor>com.example.CustomDataConvertor</dataConvertor>
    </upload>
  </excel>
</websquare>
```

**2) Java 클래스 구현**

`ICellDataProvider` 인터페이스를 구현한 Java 클래스를 작성하여, 업로드 시 `\'` → `'`로 역치환하는 로직을 추가합니다. 구현 후 WAS 클래스패스에 배치하고 재기동합니다.

- **장점**: 작은따옴표만 선택적으로 처리 가능, 다른 데이터 정제는 유지
- **주의**: Java 개발 및 WAS 재기동 필요

## 추가 확인 사항

- 두 가지 방법 모두 websquare.xml 설정 변경이 포함되어 있어 **WAS 재기동이 필요**합니다.
- WAS 재기동이 부담되는 경우, 방법 1(dataRefine=false)이 가장 간단하며, 해당 속성 추가 이후 별다른 부작용 사례는 보고되지 않았습니다.
- 기술적으로 위 두 가지 외 추가적인 수정 방향은 없는 상황입니다.
- 관련 지라: WSQ-4058

감사합니다.
