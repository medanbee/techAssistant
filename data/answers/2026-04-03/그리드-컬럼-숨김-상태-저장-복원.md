인스웨이브 기술지원팀 이단비 프로입니다.

그리드뷰 컬럼 숨김 상태 저장 및 복원과 관련하여 확인 후 답변드립니다.

## 결론

해당 기능은 WebSquare에서 공식 지원하는 **"그리드뷰 개인화"** 기능으로 구현 가능합니다. 핵심 API는 `getCurrentGridStyle()`과 `setGridStyle()`입니다.

---

## 방법 1: getCurrentGridStyle / setGridStyle (권장)

`getCurrentGridStyle()`은 현재 그리드의 **컬럼 순서, 컬럼 너비, 숨겨진 컬럼, 고정 컬럼** 상태를 문자열로 반환합니다. 이 값을 DB나 localStorage에 저장했다가, 화면 로드 시 `setGridStyle()`로 복원하면 됩니다.

### 저장 로직

```javascript
// 1. 현재 그리드 상태를 문자열로 가져오기
var gridStr = gridView1.getCurrentGridStyle();

// 2. DB에 저장 (메뉴ID, 사용자코드, 그리드명과 함께)
var saveData = {
    menuId: "MENU_001",
    userId: "USER_001",
    gridId: "gridView1",
    gridStyle: gridStr
};
// submission 등을 통해 서버로 전송하여 DB 저장
```

### 복원 로직 (화면 로드 시)

```javascript
// onpageload 또는 조회 완료 콜백에서 실행

// 1. DB에서 저장된 그리드 스타일 문자열 조회
var gridStr = /* DB에서 조회한 gridStyle 값 */;

// 2. 그리드에 적용
if (gridStr) {
    // SP4 이하: XML 파싱 후 적용
    gridView1.setGridStyle(WebSquare.xml.parse(gridStr));

    // AI 엔진(최신): JSON 문자열 직접 적용
    // gridView1.setGridStyle(gridStr);
}
```

> **참고**: `getCurrentGridStyle()`이 반환하는 형식이 엔진 버전에 따라 XML 또는 JSON으로 다를 수 있습니다. SP4에서는 `WebSquare.xml.parse()`로 파싱 후 전달해야 하며, AI 엔진(최신)에서는 JSON 문자열을 직접 전달합니다.

## 방법 2: setColumnVisible로 개별 컬럼 제어

DB에 숨김 컬럼 목록만 저장하고, 화면 로드 시 반복문으로 개별 적용하는 방식입니다.

### 저장 로직

```javascript
// 전체 컬럼을 순회하며 숨김 상태 수집
var hiddenColumns = [];
var colCount = gridView1.getColumnCount();

for (var i = 0; i < colCount; i++) {
    if (!gridView1.getColumnVisible(i)) {
        hiddenColumns.push(gridView1.getColumnID(i));
    }
}

// DB에 저장 (예: "col1,col3,col7" 형태)
var saveData = {
    menuId: "MENU_001",
    userId: "USER_001",
    gridId: "gridView1",
    hiddenColumns: hiddenColumns.join(",")
};
```

### 복원 로직 (화면 로드 시)

```javascript
// DB에서 조회한 숨김 컬럼 목록
var hiddenColumns = /* "col1,col3,col7" */;

if (hiddenColumns) {
    var cols = hiddenColumns.split(",");
    for (var i = 0; i < cols.length; i++) {
        gridView1.setColumnVisible(cols[i], false);
    }
}
```

### setColumnVisible API 명세

```
gridView.setColumnVisible(colIndex, colVisibleFlag)
```

| 파라미터 | 타입 | 필수 | 설명 |
|----------|------|------|------|
| colIndex | Number \| String | Y | 컬럼 index 또는 컬럼 id |
| colVisibleFlag | Boolean | Y | true: 표시, false: 숨김 |

## 우클릭 컨텍스트 메뉴로 Hide/Show 하는 방법

그리드뷰 속성에 `contextMenu="true"`를 설정하면, 사용자가 그리드 컬럼 body 영역에서 **마우스 우클릭 → Hide/Show**로 컬럼을 숨기거나 표시할 수 있습니다.

```xml
<w2:gridView id="gridView1" contextMenu="true" columnMove="true" ...>
```

- `contextMenu="true"` : 우클릭 시 컬럼 Hide/Show 메뉴 표시
- `columnMove="true"` : 헤더 드래그로 컬럼 순서 변경 가능

이렇게 사용자가 조정한 상태를 `getCurrentGridStyle()`로 저장하면 됩니다.

---

## 전체 흐름 요약 (권장 구현)

```
[화면 로드]
  → DB에서 사용자별 그리드 스타일 조회
  → 값이 있으면 setGridStyle()로 복원
  → 데이터 조회

[사용자 조작]
  → 우클릭으로 컬럼 Hide/Show (contextMenu="true")
  → 또는 별도 팝업에서 체크박스로 컬럼 선택

[저장 버튼 클릭 또는 화면 종료 시]
  → getCurrentGridStyle()로 현재 상태 추출
  → DB에 저장 (메뉴ID + 사용자코드 + 그리드ID + 스타일 문자열)
```

## 추가 확인 사항

1. **엔진 버전 확인**: `getCurrentGridStyle()` 반환 형식이 버전에 따라 다르므로, 사용 중인 WebSquare 엔진 버전을 확인해 주시기 바랍니다.
2. **localStorage vs DB**: 간단한 구현은 `$p.local.setItem()` / `$p.local.getItem()`으로 브라우저 localStorage에 저장할 수도 있지만, 사용자가 다른 PC에서도 동일한 설정을 유지하려면 DB 저장을 권장합니다.
3. **MDI 환경**: WindowContainer(MDI) 사용 시 창을 닫을 때 저장하는 방식으로 구성하면 자연스럽습니다.
