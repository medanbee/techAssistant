# W-Tech 게시판 기술문의 AI 자동화

> W-Tech QNA 게시판에 AI 자동 답변 기능을 도입하여 기술지원 업무를 자동화하는 프로젝트

---

## 1. 프론트엔드 (W-Tech 게시판 UI)

> W-Tech 게시판 UI에 AI 답변 및 사용자 인터랙션 기능 추가

### 기능 추가 리스트

- **Q&A 게시글 등록 페이지 개선**
  - 제목, 웹스퀘어 버전, 회사명/프로젝트 입력 필드 추가
  - 유지보수 계약 여부 확인 안내 문구 추가 (계약되어 있어야 답변 진행 가능)
- **회사명/프로젝트 검색 기능**
  - 기술지원 시스템 DB에서 유지보수 계약 확인 (지원 날짜 기준)
  - [기타] 카테고리 선택 시 AI 자동 답변 대상에서 제외
- **게시글 형식 수정**
- **AI 답변 영역 UI 추가**
  - 답변 내용 표시 (답변 신뢰도 함께 표시)
  - "엔지니어 답변 추가 요청" 버튼
  - 문의자 답글에 대한 AI 재답변 처리 (방식 검토 필요)
  - 새로 등록되는 글부터 적용 (기존 글 처리 방안 검토 필요)
- **라이선스 발급**
  - 라이선스 발급 UI 추가 → 데모 라이선스 자동 발급 기능

## 2. 서버/백엔드 (API 서버)

> AI 답변 생성 및 W-Tech와의 연동을 처리하는 API 서버 구성

### 처리 흐름

```
W-Tech 게시글 등록
  → API 서버 (TechAssistant)
  → RAG 벡터 검색 (ChromaDB + multilingual-e5-base)
  → Claude API 호출 (답변 생성)
  → API 검증 (답변 내 API/이벤트/속성명 존재 확인)
  → 검수 큐 적재
  → 담당자 승인
  → W-Tech에 답변 등록
```

### 현재 구현 상태

| API 엔드포인트 | 기능 | 상태 |
|----------------|------|------|
| `POST /api/scan` | W-Tech 신규 미답변 문의 감지 + 자동 답변 생성 (SSE) | 구현 완료 |
| `GET /api/queue` | 검수 큐 조회 (pending/approved/rejected) | 구현 완료 |
| `PATCH /api/queue/:id` | 답변 수정 및 상태 변경 (승인/반려) | 구현 완료 |
| `POST /api/answer` | 단건 답변 생성 (인증 필요) | 구현 완료 |
| `POST /api/search` | RAG 유사 사례 검색 | 구현 완료 |
| `GET /api/health` | 헬스 체크 | 구현 완료 |
| W-Tech 답변 등록 API | 승인된 답변을 W-Tech 게시판에 등록 | **미구현** |

### 미구현/검토 필요 항목

- W-Tech 답변 등록 API (W-Tech 사이트 개발 시 함께 구현 예정)
- 답글에 대한 AI 재답변 처리 방식
- 기존 게시글 소급 적용 여부 및 방법

## 3. AI 답변 생성

> 기술문의 데이터를 기반으로 AI가 답변을 생성하고, 유사 질문 및 참조 사례를 함께 제공

### 데이터 현황 (2026-04-13 기준)

| 소스 | 건수 |
|------|------|
| W-Tech QNA | 8,159건 |
| API 가이드 (AI/SP4/SP5) | 11,136건 |
| Gmail 기술문의 | 4,195건 |
| 개발가이드 샘플 | 1,157건 |
| Confluence (전체) | 1,263건 |
| 릴리즈 노트/기타 가이드 | 1,778건 |
| **합계** | **27,688건** |

- 벡터 DB: ChromaDB (multilingual-e5-base, 768차원)
- 매주 월요일 09시 자동 크롤링 (Gmail/W-Tech/Confluence 증분 수집)
- 인덱싱: 증분 방식 (신규 데이터만 임베딩)

### 답변 생성 파이프라인

1. 문의 내용 분류 (13개 대분류, 정규식 기반)
2. RAG 검색 (유사 사례 상위 8건)
3. Claude API로 답변 초안 생성
4. API 검증 (답변 내 API/이벤트/속성명을 RAG DB에서 확인)
5. 미확인 API 발견 시 재생성 (최대 3회)
6. 검수 큐 적재 → 담당자 승인

### 개인정보 처리

크롤링 시점에 `masking.js`로 즉시 마스킹 처리:

| 대상 | 치환 결과 |
|------|-----------|
| 이메일 주소 | `[이메일]` |
| 전화번호 (한국 형식) | `[전화번호]` |
| IP 주소 (포트 포함) | `[서버주소]` |
| 외부 URL (websquare/inswave/w-tech 제외) | `[URL]` |

> **참고**: 이름, 회사명, 프로젝트명은 현재 정규식으로 자동 마스킹되지 않음. 보강 필요.

### 첨부파일

- Gmail 크롤링 시 첨부파일 자동 수집 (`data/raw/gmail_attachments/`, 286건)
- 현재 RAG 인덱싱에는 포함되지 않음 (파일 시스템 저장만)
- 향후 이미지 OCR, PDF 텍스트 추출 등으로 인덱싱 포함 검토
