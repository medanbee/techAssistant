gridView 특정 셀의 편집 제어와 관련하여 확인 후 답변드립니다.

1. 원인 분석

gridView의 셀 편집 가능 여부는 컬럼 단위의 readOnly 속성 또는 셀 단위의 동적 제어를 통해 설정할 수 있습니다.

2. 해결 방법

oncellclick 이벤트에서 조건에 따라 setCellReadOnly API를 사용하여 셀 단위로 편집을 제어할 수 있습니다.

// gridView의 oncellclick 이벤트
var rowIndex = info.rowIndex;
var colId = info.columnId;
var status = gridView1.getCellData(rowIndex, "status");

if (status === "확정") {
    gridView1.setCellReadOnly(rowIndex, colId, true);
} else {
    gridView1.setCellReadOnly(rowIndex, colId, false);
}

3. 추가 확인 사항

  • setCellReadOnly는 WebSquare 5.0_3.5 이상에서 지원됩니다.
  • 행 전체를 읽기 전용으로 설정하려면 setRowReadOnly API를 사용하실 수 있습니다.

추가 문의에 대한 보완 답변입니다

Beta

추가 문의하신 setColumnProp을 활용한 컬럼 단위 readOnly 제어 방법에 대해 안내드립니다.

1. setColumnProp 사용 방법

setColumnProp API를 사용하면 컬럼 전체의 속성을 한번에 변경할 수 있습니다. 다만, 이 방법은 해당 컬럼의 모든 행에 적용되므로 특정 조건의 행만 제어하려면 setCellReadOnly가 더 적합합니다.

// 컬럼 전체를 readOnly로 설정
gridView1.setColumnProp("amount", "readOnly", true);

// 조건부 컬럼 제어가 필요한 경우
var rowCount = gridView1.getRowCount();
for (var i = 0; i < rowCount; i++) {
    var status = gridView1.getCellData(i, "status");
    if (status === "확정") {
        gridView1.setCellReadOnly(i, "amount", true);
    }
}

2. 두 API 비교

API적용 범위용도
setColumnProp컬럼 전체 (모든 행)컬럼 일괄 제어
setCellReadOnly특정 셀 (행+컬럼)조건부 셀 제어

3. 추가 확인 사항

  • setColumnProp 적용 후 개별 셀에 setCellReadOnly를 사용하면, 셀 단위 설정이 우선합니다.
  • 조건부 제어가 목적이면 이전 답변의 setCellReadOnly 방식을 권장드립니다.

✓ 채택됨

브라우저에서 마우스 우클릭 시 표시되는 컨텍스트 메뉴는 웹스퀘어가 아닌 브라우저에서 제공하는 contextmenu 이벤트를 통해 제어할 수 있습니다.

preventDefault()를 호출하면 우클릭 메뉴가 표시되지 않도록 설정할 수 있습니다.

// 전체 페이지 우클릭 방지
window.addEventListener("contextmenu", function(e) {
    e.preventDefault();
});

// 특정 요소만 우클릭 방지
var el = document.getElementById("myElement");
el.addEventListener("contextmenu", function(e) {
    e.preventDefault();
});

WEBSQUARE_DEV_PACK(SP5)은 해당 홈페이지 상단의 교육신청/일정 → 교육 예약 확인/변경/취소 → 하단의 DEVPACK download을 통해 다운로드하실 수 있습니다.
교육용 개발 라이선스는 WEBSQUARE_DEV_PACK 안에 포함되어 있는 점 참고 부탁드립니다.
