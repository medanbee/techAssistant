# 답변 편집 모달 — 설계 스펙

## 개요

AI 답변 생성 후 사용자가 직접 답변을 수정할 수 있는 모달 에디터를 추가한다. 답변 생성 직후 화면과 검수 큐 모달 양쪽에서 모두 사용 가능하다.

## 동기

현재 AI 답변은 읽기 전용으로만 표시된다. 기술지원 답변은 고객에게 전달되기 전 담당자가 내용을 보정해야 하는 경우가 많다. 답변 복사 → 외부 편집 → 다시 붙여넣기 워크플로우를 제거하고, 시스템 내에서 직접 수정 + 미리보기 + 저장까지 완결한다.

## UI 방식: 모달 에디터

별도 모달이 전체 화면 위에 뜨는 방식. 넓은 편집 공간과 편집/미리보기 분할 뷰를 제공한다.

## 편집 모달 진입 지점 (2곳)

### 1. AI 답변 섹션 (`#answerSection`)
- 기존 "답변 복사" / "검수 큐로 보내기" 버튼 옆에 **"답변 수정"** 버튼 추가
- 클릭 시 편집 모달이 열림
- 저장 시 `answerContent`와 `_lastAnswer.answer`가 수정된 내용으로 갱신

### 2. 검수 큐 모달 (`#queueModal`)
- 기존 "답변 복사" 버튼 옆에 **"답변 수정"** 버튼 추가 (pending 상태일 때만)
- 클릭 시 편집 모달이 열림 (검수 큐 모달 위에 중첩)
- 저장 시 `PATCH /api/queue/:id` body에 `{ answer: 수정된답변 }`으로 서버 업데이트
- 모달 내 답변 표시도 수정된 내용으로 갱신

## 모달 레이아웃

```
┌─────────────────────────────────────────────────┐
│ [✏️ 답변 수정]   [Diff 보기] [↩ 원본 복원]  [✕] │  ← 헤더
├────────────────────┬────────────────────────────┤
│     편집 (MD)      │      미리보기 (렌더링)       │  ← 분할 뷰
│                    │                            │
│  마크다운 텍스트     │  실시간 렌더링 결과           │
│  직접 입력/수정      │  (marked.js)              │
│                    │                            │
├────────────────────┴────────────────────────────┤
│ [Ctrl+S 저장] [Esc 닫기] [Ctrl+Z]   [취소] [저장] │  ← 푸터
└─────────────────────────────────────────────────┘
```

### 헤더
- 좌측: 아이콘 + "답변 수정" 제목 + "마크다운 편집 · 실시간 미리보기" 서브텍스트
- 우측: Diff 보기 토글, 원본 복원 버튼, 닫기(✕) 버튼

### 본문 — 분할 뷰
- **왼쪽: 편집 패널** — `<textarea>`에 마크다운 원문 표시. 사용자가 직접 수정.
- **오른쪽: 미리보기 패널** — `marked.js`로 실시간 렌더링. 편집 패널의 `input` 이벤트에 반응하여 즉시 갱신.
- 모바일(≤640px)에서는 탭 전환 방식 (편집/미리보기 탭)

### 푸터
- 좌측: 키보드 단축키 안내 텍스트
- 우측: 취소 버튼(ghost) + 저장 버튼(primary)

## 핵심 기능

### 1. 실시간 미리보기
- 편집 패널의 `input` 이벤트마다 `marked.parse()`로 미리보기 갱신
- 성능: `requestAnimationFrame`으로 throttle (빠른 타이핑 대응)

### 2. 원본 복원
- 모달 오픈 시 원본 텍스트를 `_originalAnswer` 변수에 별도 저장
- "원본 복원" 클릭 → `confirm('수정한 내용이 사라집니다. 원본으로 복원할까요?')` → 확인 시 편집 패널을 원본으로 교체
- 편집 내용이 원본과 동일하면 버튼 비활성화 (회색 처리)

### 3. 변경사항 표시 (Diff)
- "Diff 보기" 토글 ON 시 미리보기 패널 대신 diff 패널로 전환
- 구현: 간단한 라인 단위 diff 알고리즘 (외부 라이브러리 없이)
  - 원본/수정본을 줄 단위로 비교
  - 삭제된 줄: 빨간 배경 (`#fef2f2`, 텍스트 `#b91c1c`)
  - 추가된 줄: 초록 배경 (`#ecfdf5`, 텍스트 `#166534`)
  - 동일한 줄: 기본 배경
- Diff 보기 OFF 시 다시 미리보기 패널로 전환

### 4. 키보드 단축키
- `Ctrl+S` (또는 `Cmd+S`): 저장 후 모달 닫기
- `Esc`: 변경 사항 있으면 확인 팝업 → 모달 닫기
- `Ctrl+Z` / `Ctrl+Shift+Z`: 브라우저 기본 undo/redo (textarea 네이티브)

## 데이터 흐름

### AI 답변 섹션에서 수정
```
수정 버튼 클릭
  → 모달 오픈 (원본 = _lastAnswer.answer)
  → 사용자 편집
  → 저장 클릭
  → _lastAnswer.answer = 수정본
  → answerContent.innerHTML = renderMarkdown(수정본)
  → 모달 닫기
```

### 검수 큐에서 수정
```
수정 버튼 클릭
  → 모달 오픈 (원본 = _currentItem.answer)
  → 사용자 편집
  → 저장 클릭
  → PATCH /api/queue/:id  { answer: 수정본 }
  → _currentItem.answer = 수정본
  → modalAnswer.innerHTML = renderMarkdown(수정본)
  → 편집 모달 닫기
```

## 서버 변경

### `PATCH /api/queue/:id`
기존 `{ status }` body에 `answer` 필드 추가 허용:
- `{ status: 'approved' }` — 기존 승인
- `{ answer: '수정된 답변' }` — 답변만 수정 (status 불변)
- `{ status: 'approved', answer: '수정된 답변' }` — 수정 + 승인 동시

## 새로 추가되는 파일/요소

### HTML (`index.html`)
- `#editModal` — 편집 모달 마크업 (body 끝에 추가)

### CSS (`style.css`)
- `.edit-modal` 관련 스타일 (모달, 분할 뷰, diff 하이라이트)
- 반응형: ≤640px에서 분할 뷰 → 탭 전환

### JS (`app.js`)
- `openEditModal(source)` — source: 'answer' | 'queue'. 모달 오픈 + 원본 저장
- `closeEditModal()` — 변경 확인 + 모달 닫기
- `saveEdit()` — 저장 처리 (소스에 따라 분기)
- `revertToOriginal()` — 원본 복원
- `toggleDiff()` — diff 뷰 토글
- `simpleDiff(original, modified)` — 라인 단위 diff 계산

## 반응형 동작 (모바일)

- 모달 크기: `width: 96%; max-height: 92vh;`
- 헤더 버튼들 아이콘만 표시 (텍스트 숨김)
