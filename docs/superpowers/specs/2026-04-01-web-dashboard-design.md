# 기술문의 자동 답변 시스템 — 웹 대시보드 + W-Tech 자동화 설계

## 배경

현재 기술문의 답변 시스템은 CLI(Claude Code)에서 수동으로 운영 중이다.
- 담당자 1명만 사용 가능 (CLI 진입장벽)
- RAG 검색 → 답변 생성 → 복사 → W-Tech 붙여넣기 수작업
- 답변 품질 검증이 수동 (잘못된 API 추천 위험)

## 목표

1. 팀원 누구나 웹 브라우저에서 답변 생성 가능
2. W-Tech 신규 문의 자동 감지 + 답변 초안 생성
3. 사람 검수 후 W-Tech 자동 등록
4. 단계적 도입 — 각 단계 완료 시 바로 실무 투입

## 단계별 설계

### 1단계: REST API 서버

기존 pipeline.js를 HTTP 엔드포인트로 감싸는 핵심 서버.

#### 엔드포인트

| Method | Path | 설명 |
|--------|------|------|
| `POST` | `/api/answer` | 문의 → 답변 생성 (분류→RAG→생성→API검증) |
| `POST` | `/api/search` | RAG 검색만 (유사 사례 조회) |
| `GET`  | `/api/health` | 서버 상태 확인 |

#### 요청/응답

```
POST /api/answer
Request:  { "question": "gridView 셀 병합 방법", "version": "v5.0" }
Response: { "answer": "...", "sources": [...], "category": "...", "verified": true }
```

#### 기술 선택

- **Express** — 기존 Node.js 프로젝트, 추가 의존성 최소화
- 기존 `pipeline.js`, `answerGenerator.js`, `searcher.py` 그대로 재활용
- **SSE(Server-Sent Events)** — 답변 생성 진행 상태 실시간 전달 (분류중 → RAG검색중 → 생성중 → 검증중)
- **인증** — 내부망 전용, 간단한 API 키 인증 (config.json에 설정)

#### 파일 구조

```
src/api/
  server.js        — Express 서버 진입점
  routes/
    answer.js      — /api/answer 라우트
    search.js      — /api/search 라우트
  middleware/
    auth.js        — API 키 인증
```

### 2단계: 웹 폼 UI

API 서버 위에 팀원 누구나 쓸 수 있는 웹 화면.

#### 화면 구성

```
┌─────────────────────────────────────┐
│  기술문의 답변 시스템                    │
├─────────────────────────────────────┤
│  [문의 내용 입력 textarea]              │
│  [버전 선택 ▾]  [답변 생성 버튼]         │
├─────────────────────────────────────┤
│  답변 초안                             │
│  ┌─────────────────────────────────┐│
│  │ 생성된 답변 내용 (편집 가능)        ││
│  └─────────────────────────────────┘│
│  참고 출처 (RAG 결과 목록)              │
│  [복사] [수정 후 복사] [답변 저장]       │
└─────────────────────────────────────┘
```

#### 핵심 기능

- 문의 붙여넣기 → 답변 생성 → 편집/복사 → W-Tech에 수동 붙여넣기
- SSE로 단계별 진행률 표시
- 생성된 답변 이력 조회

#### 기술

- **바닐라 HTML/CSS/JS** — 정적 파일, Express에서 `public/` 폴더로 서빙
- WebSquare 팀 특성상 무거운 프레임워크보다 유지보수 용이

#### 파일 구조

```
public/
  index.html       — 메인 페이지
  css/style.css    — 스타일
  js/app.js        — 클라이언트 로직
```

### 3단계: W-Tech 자동 모니터링 + 검수 큐

W-Tech 신규 문의를 자동 감지하고, 답변 초안을 만들어 검수 큐에 넣는 단계.

#### 흐름

```
W-Tech 크롤러 (10분 간격)
  → 신규 문의 감지
  → API 서버로 답변 생성 요청
  → 검수 큐에 "대기" 상태로 저장
  → 웹 대시보드에서 검수
  → 승인 시 Puppeteer로 W-Tech 자동 등록
```

#### 검수 큐 상태

```
대기(pending) → 승인(approved) → 등록완료(posted)
                → 반려(rejected)
```

#### 대시보드 확장

2단계 웹 UI에 탭 추가:
- **수동 입력 탭** — 기존 폼 UI
- **검수 큐 탭** — 자동 생성된 답변 목록, 승인/반려/수정 기능

#### 기술

- **크롤러**: 기존 `wtechCollector.js` 재활용, 신규 글 감지 모드 추가
- **검수 큐 DB**: SQLite (`data/queue.db`) — 상태, 문의 내용, 답변 초안, 타임스탬프
- **자동 등록**: Puppeteer로 W-Tech 로그인 → 답변 작성 → 등록
- **스케줄러**: node-cron으로 10분 간격 실행

#### 안전장치

- 반드시 사람 승인 후에만 W-Tech 등록
- 등록 전 미리보기 제공
- 등록 실패 시 재시도 + 상태 기록
- 나중에 신뢰도 기반 자동 승인으로 전환 가능

#### 파일 구조

```
src/api/
  routes/
    queue.js       — 검수 큐 CRUD 라우트
  services/
    monitor.js     — W-Tech 신규 문의 모니터링
    poster.js      — W-Tech 답변 자동 등록 (Puppeteer)
  db/
    queue.js       — SQLite 검수 큐 모델
```

## 전체 아키텍처

```
                    ┌──────────────┐
                    │  웹 브라우저   │
                    │  (팀원 접속)   │
                    └──────┬───────┘
                           │ HTTP
                    ┌──────▼───────┐
                    │ Express 서버  │
                    │  /api/*      │
                    │  /public/*   │
                    └──┬───┬───┬──┘
                       │   │   │
          ┌────────────┘   │   └────────────┐
          ▼                ▼                ▼
    ┌──────────┐   ┌──────────────┐  ┌───────────┐
    │pipeline.js│   │ 검수 큐 DB   │  │ W-Tech    │
    │searcher.py│   │ (SQLite)     │  │ 모니터링   │
    │(기존 코드) │   └──────────────┘  │ + 등록     │
    └──────────┘                     │(Puppeteer) │
                                     └───────────┘
```

## 기술 스택 요약

- **서버**: Express (Node.js)
- **프론트**: 바닐라 HTML/CSS/JS (정적 서빙)
- **DB**: SQLite (검수 큐)
- **자동화**: Puppeteer (W-Tech 크롤링/등록), node-cron (스케줄러)
- **기존 재활용**: pipeline.js, answerGenerator.js, searcher.py, wtechCollector.js

## 원칙

- 각 단계 완료 시점마다 실무 투입 가능
- 답변 자동 등록은 반드시 사람 검수 후 (추후 자동 승인 전환 가능)
- 내부망 전용, API 키 인증
- 기존 코드 최대한 재활용, 새 의존성 최소화
