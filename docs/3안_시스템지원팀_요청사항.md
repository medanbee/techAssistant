# 3안(W-Tech 고객 직접 응답) 구현을 위한 시스템지원팀 협조 요청 사항

> 작성일: 2026-04-09

---

## 1. 개요

기술문의 자동 답변 시스템 3안(W-Tech 고객 직접 응답)을 구현하기 위해 시스템지원팀의 협조가 필요한 항목을 정리합니다.

### 3안 동작 방식

```
고객 W-Tech 문의 등록 → AI 즉시 답변 생성(RAG+Claude) → 고객에 표시(답변+신뢰도+참조사례) → 고객 선택(채택/추가요청/부분) → 후속 처리
```

## 2. 서버 환경 구성 요청

### 2-1. API 서버 (1대)

| 항목     | 요구사항                                            |
| -------- | --------------------------------------------------- |
| OS       | Linux (CentOS/Ubuntu) 또는 Windows Server           |
| CPU      | 4코어 이상 (임베딩 모델 연산용)                     |
| RAM      | 8GB 이상 (ChromaDB + 임베딩 모델 상주)              |
| 디스크   | 20GB 이상 (벡터DB + 데이터)                         |
| 네트워크 | 인트라넷 접근 가능, 외부 API 호출 가능 (Claude API) |
| 포트     | 3000번 (API 서버) 오픈 필요                         |

### 2-2. 필요 소프트웨어

| 소프트웨어         | 버전      | 용도                                        |
| ------------------ | --------- | ------------------------------------------- |
| Node.js            | 18 이상   | API 서버 (Express)                          |
| Python             | 3.10 이상 | RAG 검색 (ChromaDB + sentence-transformers) |
| PM2 (또는 systemd) | 최신      | 프로세스 관리 (자동 재시작)                 |
| Git                | 최신      | 소스 배포                                   |

### 2-3. 외부 통신

| 대상          | URL               | 용도                    | 비고                                                                                                                                                                   |
| ------------- | ----------------- | ----------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Anthropic API | api.anthropic.com | Claude AI 답변 생성     | HTTPS 443, 상시 필요                                                                                                                                                   |
| HuggingFace   | huggingface.co    | AI 임베딩 모델 다운로드 | HTTPS 443, 최초 세팅 시 1회만 필요 (이후 로컬 캐시 사용). 텍스트를 벡터로 변환하는 검색용 AI 모델(multilingual-e5-base, 약 1.1GB)을 다운로드하는 오픈소스 AI 모델 허브 |

## 3. W-Tech 연동 요청

### 3-1. W-Tech DB 접근

| 항목        | 상세                                          |
| ----------- | --------------------------------------------- |
| 목적        | 신규 문의 감지 + AI 답변 게시                 |
| 필요 테이블 | QNA 문의/답변 테이블 (조회 + 쓰기)            |
| 접근 방식   | DB 직접 접근 또는 W-Tech 내부 API 제공        |
| 필요 권한   | 문의 조회 (SELECT), 답변 등록 (INSERT/UPDATE) |

### 3-2. W-Tech 포털 UI 수정

AI 답변 표시를 위해 W-Tech QNA 화면에 아래 UI 요소 추가가 필요합니다.

| UI 요소      | 설명                                              |
| ------------ | ------------------------------------------------- |
| AI 답변 영역 | 문의 상세 페이지에 AI 자동 답변 표시 영역         |
| 면책 고지    | "이 답변은 AI가 자동 생성한 내용입니다" 안내 배너 |
| 신뢰도 표시  | RAG 검색 유사도 기반 신뢰도 바 (%)                |
| 피드백 버튼  | 채택 / 추가 요청 / 부분 도움 (3개 버튼)           |
| 코드 블록    | 코드 구문 하이라이팅 + 복사 기능                  |

### 3-3. 연동 방식 (택 1)

**방식 A: API 연동 (추천)**

- W-Tech에서 문의 등록 시 기술지원팀 API 서버로 webhook 호출
- API 서버가 AI 답변 생성 후 응답 반환
- W-Tech에서 응답 받아 화면에 표시

**방식 B: DB 직접 연동**

- API 서버가 주기적으로 W-Tech DB 폴링하여 신규 문의 감지
- AI 답변 생성 후 W-Tech 답변 테이블에 직접 INSERT

## 4. 보안 관련

### 4-1. 데이터 마스킹 등급

현재 답변 생성 시 마스킹 처리가 적용되어 있으며, 아래 등급을 따릅니다.

| 등급          | 대상                            | 처리        |
| ------------- | ------------------------------- | ----------- |
| Red (높음)    | 라이선스 키, IP, 토큰, 비밀번호 | 완전 마스킹 |
| Yellow (중간) | 서버 구성, 내부 URL, 프로젝트명 | 부분 마스킹 |
| Green (낮음)  | 에러코드, 설정값, API명         | 그대로 활용 |

### 4-2. API 인증

- API 서버는 X-API-Key 헤더 기반 인증 적용
- W-Tech 연동 시 전용 API Key 발급 필요

### 4-3. Claude API Key 관리

- 공용 API Key 사용 (개인 ID + 공용 Key)
- 서버 환경변수 또는 config 파일로 관리
- 월간 비용 예상: ~$50 (200건/월 기준)
